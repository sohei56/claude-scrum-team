# Implementation Plan: AI-Powered Scrum Team

**Branch**: `001-ai-scrum-team` | **Date**: 2026-03-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-ai-scrum-team/spec.md`

## Summary

Build a shell-script-launched AI-powered Scrum development team for Claude Code.
The system uses Claude Code Agent Teams (experimental) with the Scrum Master as
team lead **in Delegate mode** (coordination only — cannot write code) and
Developer teammates spawned per Sprint via the `spawn-teammates` Skill. A
Textual-based TUI dashboard runs alongside Claude Code in a tmux pane,
displaying Sprint Overview, PBI Progress Board, Communication Log, and File
Change Log in real time via watchdog filesystem monitoring. All state persists
as JSON files in `.scrum/`. Scrum ceremonies are encapsulated as Skills with
mandatory Inputs/Outputs declarations. Design documents are governed by
`.design/catalog.md` — no design document may be created unless its spec type
is listed and enabled in the catalog. Files live at
`.design/specs/{category}/{id}-{slug}.md`. Sub-agent selection from the
awesome-claude-code-subagents catalog is reproducible via the `install-subagents`
Skill.

## Technical Context

**Language/Version**: Bash 3.2+ (shell scripts, hooks), Python 3.9+ (TUI dashboard), Markdown with YAML frontmatter (agent/skill definitions)
**Primary Dependencies**: Claude Code CLI with Agent Teams (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` set process-scoped by `scrum-start.sh`), Textual 8.x (TUI framework), watchdog (filesystem monitoring)
**Storage**: JSON files in `.scrum/` directory (one file per concern), Markdown for requirements and design documents (governed by `.design/catalog.md`)
**Testing**: bats-core + jq + yq + ShellCheck (developer-only dependencies)
**Target Platform**: macOS / Linux terminal (Bash 3.2+ default)
**Project Type**: single
**Performance Goals**: Sub-second dashboard updates via watchdog filesystem events; 1-6 concurrent Developer teammates
**Constraints**: Bash 3.2+ compatibility (macOS default shell), minimal end-user dependencies (Claude Code + Python 3.9+ + pip packages); venv recommended but not required
**Scale/Scope**: 1-6 Developer agents per Sprint, no global PBI limit (refined PBI WIP capped at 1-2 Sprints of capacity), design documents governed by `.design/catalog.md` with tracked revision history
